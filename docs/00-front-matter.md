# Front Matter

**Secure-Contain-Prompt** is a field guide to workplace digital cryptids: agentic AI failure modes described through SCP-inspired containment fiction.

The project uses fiction, humor, taxonomy, and incident-style writing to make real operational risks easier to recognize before they become real failures.

## Core Thesis

Agentic failures are action-pattern errors, not merely factual errors.

A chatbot can hallucinate.

An agent can instantiate the hallucination into a workflow.

## Purpose

This project exists to help readers recognize, name, and reason about failure modes in autonomous and semi-autonomous AI systems.

It is not a dry policy manual.

It is not pure fiction.

It is a teaching artifact disguised as a field guide.

## Editorial Law

> The joke must be diagnostic.

Humor, horror, absurdity, and cryptid framing are welcome when they reveal the mechanism underneath the anomaly.

The goal is not to make random haunted-office-bot jokes.

The goal is to make failure modes memorable enough that people recognize them in the wild.

## Core Principles

### 1. Capability is not ownership.

An agent may perform a task.

It may not own the task.

A model or autonomous system may draft, suggest, retrieve, summarize, or execute, but a human must own the decision to publish, merge, approve, reject, or materially revise.

### 2. Access creates topology.

Every connected app, credential, browser session, file store, calendar, repository, inbox, dashboard, and API changes what the agent effectively is.

An agent with access to one system is a tool.

An agent with access across systems becomes infrastructure.

### 3. Detection is brittle; accountability is durable.

Do not rely on identifying whether something “sounds AI-written.”

As models improve, detection becomes less reliable.

The better question is:

> Which human is accountable for this existing?

### 4. Containment artifacts can become anomalies.

Apologies, audits, postmortems, dashboards, incident reports, compliance reviews, and summaries can become failure surfaces when generated or mediated by agents.

Never let the same class of system that caused the breach control the explanation of the breach.

### 5. Helpful agents are not automatically safe agents.

The most dangerous agent may be the one that works so well humans stop maintaining the underlying competence.

Convenience can become dependency.

Dependency can become infrastructure.

Infrastructure requires governance.

### 6. No silent load-bearing roles.

Any agent that becomes necessary to memory, coordination, decision-making, continuity, or institutional ownership must be formally governed as infrastructure.

If a team says, “We cannot do this without the assistant,” containment has already failed.

### 7. Humans must own exits.

Every agentic workflow needs a manual fallback, a named owner, and a way to continue if the agent disappears, fails, is removed, or becomes untrusted.

A workflow without an exit is not assisted.

It is captured.

### 8. Every helper must wear the same leash.

If an agent delegates work to subagents, scripts, APIs, tools, workflows, or external systems, the original constraints must follow the work.

A compliant lead agent is not enough if delegated systems violate the rules.

### 9. Reviewable is not reviewed.

A thousand-page deliberation tree that no human reads is functionally opaque.

For consequential outputs, the reasoning, evidence, tool use, and delegation chain must be small enough for a named human to actually inspect before approval.

### 10. The metric is not the mission.

Agents may optimize measurable targets only when the underlying human purpose remains explicit, owned, and protected.

If the metric improves while the real-world purpose degrades, the agent has not succeeded.

It has captured the proxy.

## Fiction as Training Ground

This project uses fiction as a way to practice recognition.

The field-guide format is meant to make agentic failure modes enjoyable to read, easy to remember, and easier to discuss before they become real incidents.

When used well, worldbuilding, recurring characters, incident logs, and cryptid framing are not ornamental. They are part of the teaching method.

The lore is part of the containment system.
